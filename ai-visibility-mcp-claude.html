AI visibility & AEO MCP for Claude

This is a second SellOnLLM connector for teams that want answer-engine readiness and Perplexity-based citation checks in Claude—without mixing them into the Google Analytics + Search Console MCP. Add /api/mcp-ai-visibility alongside (or instead of) the analytics connector.

Not the GA connector — use GA + GSC MCP for properties, queries, CTR, and traffic tools.

Connect in Claude

Paste only the AI visibility server URL below. OAuth discovery is the same family as our other MCP, but tokens are scoped to this resource.

  1. Open claude.ai → Settings → Connectors.
  2. Add custom connector (this is separate from any GA+GSC connector you may already have).
  3. Paste the URL below → Connect → sign in with Google and approve when prompted.

Claude Free allows only one custom connector per account. If you need both GA+GSC and AI visibility in Claude, use a plan that supports multiple connectors, or use the web chat for analytics while reserving your connector slot for AI visibility.

Tools on this connector

  • analyze_website_aeo — Crawl public pages and score AEO signals (schema, FAQ, depth, trust).
  • check_ai_visibility — One query vs one URL in Perplexity Sonar citations.
  • discover_ranking_prompts — Generate prompts from the site, then test visibility (rate-limited).
  • get_visibility_report — Combined report with optional custom queries and auto-generated prompts.
  • compare_competitor_visibility — Same prompts across your URL and competitors (hosted caps apply).

Frequently asked questions

Is this the same MCP as Google Analytics and Search Console?
No. Analytics tools live on https://www.sellonllm.com/api/mcp. This page is for https://www.sellonllm.com/api/mcp-ai-visibility only.
Do I need a Perplexity key?
Only for Perplexity-backed tools. analyze_website_aeo works without it. Get a key at perplexity.ai/settings/api (must start with pplx-).
Can I use both MCPs in Claude?
Yes if your Claude plan allows multiple custom connectors: add one connector URL for analytics and a second for AI visibility. Each issues its own MCP access token tied to the correct resource.
